## SP-5: Error Handling (Mandatory Capability)

### Failure Classes Covered

| Failure Class         | Detection              | AI Resolution Path              |
| --------------------- | ---------------------- | ------------------------------- |
| Build Failure         | Docker build logs      | `docker ai "fix build error"`   |
| Pod CrashLoopBackOff  | `kubectl get pods`     | `kubectl-ai "why pods failing"` |
| Service Unreachable   | Health check failure   | `kagent "diagnose service"`     |
| Helm Upgrade Failure  | `helm status`          | `kubectl-ai "rollback + fix"`   |

### Resolution Protocol

```
1. Detect вҶ’ AI agent monitors cluster state
2. Diagnose вҶ’ kagent/kubectl-ai analyzes root cause
3. Fix вҶ’ AI generates corrected YAML or config
4. Apply вҶ’ Helm upgrade with AI-generated fix
5. Verify вҶ’ AI confirms resolution
```

> **Manual debugging is EXPLICITLY FORBIDDEN (SP-0.2)**

## Quick Start Commands (Post-Spec)

```bash
# 1. Start Minikube
minikube start

# 2. Build Docker images in Minikube context
eval $(minikube docker-env)
docker build -f Dockerfile.frontend -t todo-frontend:latest .
docker build -f Dockerfile.backend -t todo-backend:latest .

# 3. Install via Helm
helm install todo-app ./charts/todo-app --namespace todo --create-namespace

# 4. Verify deployment
kubectl get all -n todo

# 5. Access frontend
minikube service todo-frontend --namespace todo

# 6. Scale backend (SP-1.2.F4)
kubectl scale deployment todo-backend --replicas=3 -n todo

# 7. Cleanup
helm uninstall todo-app -n todo
kubectl delete namespace todo
```
